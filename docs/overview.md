# MACR：基于多智能体协同与知识图谱的程序自动修复框架

## 1. 研究背景与动机

随着软件系统规模和复杂度的持续增长，程序缺陷（Bug）修复成本不断攀升。传统基于启发式规则或单一神经网络的自动程序修复（Automated Program Repair, APR）方法在处理跨文件、多模块耦合的复杂缺陷时，往往受限于上下文理解能力不足与修复策略单一。近年来，大语言模型（LLM）与多智能体（Multi-Agent）系统为 APR 带来了新的机遇，但现有方法仍面临两大挑战：

1. **上下文污染与一致性缺失**：单一 Agent 在长程修复过程中容易累积无关上下文，导致决策漂移；而完全独立的多个 Agent 又难以保证协同一致性。
2. **代码依赖理解不精确**：LLM 对代码库全局结构的理解有限，难以有效追踪跨文件、跨模块的依赖关系。

针对上述问题，本研究提出 MACR（Multi-Agent Collaborative Repair）框架，通过**动态角色编排**与**代码知识图谱**实现多智能体协同修复。

## 2. 研究目标

构建并验证一种面向真实代码仓库的多智能体协同自动修复框架 MACR，使其能够：

- 将复杂修复任务分解为可并行/串行执行的子任务；
- 通过动态子智能体编排避免上下文污染，同时保持全局一致性；
- 利用基于 AST 构建的代码知识图谱，为智能体提供精确的代码依赖与结构信息；
- 在经典 APR 数据集上取得具有竞争力的修复效果，并验证各核心模块的有效性。

## 3. 核心方法

### 3.1 动态角色编排（Dynamic Role Orchestration）

MACR 采用“主 Agent + 子 Agent”的分层协同架构，借鉴当前先进 LLM 编程助手（如 Codex CLI、Claude Code）的通用范式：

- **主 Agent（Coordinator）**：负责任务分解、子任务派发、结果汇总与最终修复补丁生成；
- **子 Agent（Worker）**：负责执行具体子任务（如定位缺陷、生成候选补丁、验证补丁等），并将结果返回给主 Agent；
- **上下文隔离**：每个子 Agent 只接收与其子任务相关的上下文，避免全局上下文污染；
- **一致性保证**：主 Agent 通过统一的任务协议与结果格式维护全局一致性。

### 3.2 代码知识图谱（Code Knowledge Graph）

知识图谱是本框架的重点模块，旨在增强 Agent 对代码库结构的理解：

- **图谱构建**：基于代码抽象语法树（AST）动态抽取类、方法、字段、调用关系、继承关系等实体与关系，构建可查询的代码知识图谱；
- **开放查询接口**：将图谱以统一接口开放给主 Agent 与各个子 Agent，支持按需检索相关代码实体及其依赖；
- **精确依赖追踪**：帮助 Agent 快速定位缺陷传播路径，减少无关代码干扰，提高补丁生成的准确性与可解释性。

### 3.3 模块化与可插拔设计

为支持消融实验，动态角色编排与知识图谱均被设计为**可插拔模块**：

- 可独立启用/禁用子 Agent 协同机制；
- 可独立启用/禁用知识图谱辅助；
- 便于对比不同配置下的修复性能，验证各模块的贡献。

## 4. 实验设计

### 4.1 数据集

| 数据集 | 语言 | 缺陷数量 | 特点与用途 | 适用场景 |
|--------|------|----------|------------|----------|
| Defects4J (v2.0) | Java | 835 | 来自真实开发历史，配套完整测试用例 | 验证基础修复能力与回归测试 |
| SWE-bench | Python | 2,294 | 来自 GitHub 真实 Issue，包含复杂多文件修改 | 评估解决真实复杂问题的能力 |
| QuixBugs | Python/Java | 40 | 经典算法类缺陷，规模较小 | 早期原型验证与快速迭代 |

### 4.2 评估指标

- **修复数量 / 修复率（Plausible Patches）**：通过测试用例的补丁数量；
- **正确补丁比例（Correct Patches）**：与开发者实际修复一致的补丁比例；
- **平均尝试次数 / 耗时**：衡量修复效率；
- **回归测试通过率**：验证补丁是否破坏原有功能。

### 4.3 消融实验

为验证动态角色编排与知识图谱各自的有效性，设计以下消融配置：

1. **MACR（完整框架）**：启用动态角色编排 + 知识图谱；
2. **MACR − KG**：禁用知识图谱，保留动态角色编排；
3. **MACR − Sub-Agents**：禁用子 Agent 协同，保留知识图谱；
4. **Baseline**：禁用知识图谱与子 Agent 协同，退化为单 Agent + LLM 基础方法。

## 5. 预期结果

本研究不追求在所有指标上达到当前最优（SOTA）水平，但期望 MACR 在以下方面取得亮眼表现：

- 在 SWE-bench 等复杂真实场景下，修复率显著优于单 Agent Baseline；
- 知识图谱模块能够有效提升跨文件缺陷的定位与修复能力；
- 动态角色编排机制能够在控制上下文长度的同时，保持修复结果的一致性；
- 消融实验能够清晰展示两个核心模块对整体性能的贡献。

## 6. 创新点

- 将多智能体动态角色编排与代码知识图谱相结合，提出面向 APR 的协同修复框架；
- 基于 AST 构建动态、可查询的代码知识图谱，增强 Agent 对代码库结构的理解；
- 通过模块化设计系统验证两个核心机制的贡献，提升研究的可解释性与可复现性。
